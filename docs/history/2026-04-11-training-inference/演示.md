## 2. 原理怎么讲

- 推理预测逻辑：
  先按计算图解析每层和每个模块的大致耗时，再用真实 profiling 采到的模块时间去覆盖关键模块，最后把计算时间和通信时间组合成 `prefill / decode / request` 总时间。
- 训练预测逻辑：
  把一次训练迭代拆成 `first_slot + (mb-1) * steady_slot + optimizer_step`，先实测这些组件，再把它们拼回整轮训练时间。
- 所以本质不是“训练一个黑盒模型”，而是：
  `解析估算 + JSONL profiling 查表修正 + 按并行结构重新组合`
