5.1.13 算子级建模介入与编程接口测试
（1）测试项名称
算子级建模介入与编程接口测试
（2）测试项标识
OP-LEVEL-IF-TEST
（3）测试内容
在异构计算架构性能建模工具中，验证算子级建模接口的可用性。选取LLama3.1-8B模型中的典型算子（如矩阵乘、注意力机制计算等）作为输入，通过编程接口提交算子描述，算子级模型能够返回预测执行时间，且接口调用过程无异常。
（4）测试方法
A．在搭载性能建模工具的环境（如英特尔CPU+英伟达GPU服务器）上，配置算子级建模接口所需的依赖库及运行环境。
B．从LLama3.1-8B模型中提取若干代表性算子（包括计算密集型算子和访存密集型算子），获取其参数（如输入尺寸、数据类型、并行策略等）。
C．通过算子级建模编程接口，依次提交上述算子的描述信息，调用模型预测接口，获取每个算子的预测执行时间。
D．检查接口调用是否成功、返回结果是否合理（如时间值为正数、格式正确）。
E．以接口调用无报错、返回时间值为判定标准，验证算子级建模介入与编程接口的基本功能。




寒武纪架构计算密集型算子空间维度建模测试
（1）测试项名称
寒武纪架构计算密集型算子空间维度建模测试
（2）测试项标识
CAMBRICON-COMPUTE-OP-SPACE-TEST
（3）测试内容
在寒武纪MLU硬件架构环境下，验证算子级空间维度模型对计算密集型算子的预测准确性。选取LLama3.1-8B模型中的典型计算密集型算子（如 `MLP Up GEMM`、`MLP Gate GEMM`、`MLP Down GEMM`、`FlashAttention`、`Attention Output Proj GEMM`），分别在单卡、单机两卡规模下记录实际执行时间，并与模型预测时间对比，误差不超过20%。
（4）测试方法
A．在搭载寒武纪MLU加速卡的服务器上配置性能建模工具所需环境，测试终端机连接测试服务器。
B．从LLama3.1-8B模型中提取代表性计算密集型算子（如 `MLP Up GEMM`、`MLP Gate GEMM`、`MLP Down GEMM`、`FlashAttention`、`Attention Output Proj GEMM`），确定其输入尺寸、数据类型等参数。
C．分别在单卡、单机两卡两种并行规模下，运行实际算子执行（可使用寒武纪CNNL或PyTorch微基准测试），记录每个算子的实际执行时间T_real（五次运行取平均值）。
D．使用任务响应时间分析工具的算子级空间维度模型，输入相同算子描述和并行配置，得到预测时间T_sim。
E．计算误差 = |T_real - T_sim| / T_real × 100%，记录每个算子在不同规模下的误差值。
F．以所有测试算子的误差均≤20%为判定标准，验证寒武纪架构上计算密集型算子空间维度建模的准确性。

寒武纪架构并行配置下训练任务时间维度测试
（1）测试项名称
寒武纪架构并行配置下训练任务时间维度测试
（2）测试项标识
CAMBRICON-PARALLEL-TRAIN-TIME-TEST
（3）测试内容
在寒武纪MLU硬件架构环境下，验证应用级时空双维度模型对训练任务在不同并行配置下的时间预测准确性。以LLama3.1-8B模型训练任务为例，调整流水线并行路数和微批次数量，在单机两卡规模下记录实际每迭代时间，并与模拟时间对比，误差不超过20%。
（4）测试方法
A．在搭载寒武纪MLU加速卡的服务器上配置性能建模工具所需环境、深度学习框架及训练脚本，测试终端机连接测试服务器。
B．准备LLama3.1-8B模型训练任务脚本，支持配置流水线并行路数（如1、2）和微批次数量（如1、2、4），确保能在寒武纪平台上运行。
C．在单机两卡规模下，组合不同的流水线并行路数和微批次数量（至少3种组合），运行实际训练任务，记录每个配置下稳定后的平均每迭代时间T_real（五次运行取平均值）。
D．使用任务响应时间分析工具，输入相同模型描述、并行配置和硬件拓扑，模拟得到每迭代时间T_sim。
E．计算误差 = |T_real - T_sim| / T_real × 100%，记录每种并行配置下的误差值。
F．以所有测试配置的误差均≤20%为判定标准，验证寒武纪架构上训练任务时间维度建模的准确性。
